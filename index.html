Design of High-Speed 8-bit Vedic Multiplier using Brent-Kung Adders

This page follows the structure of the published IEEE work and explains each part in language that non-VLSI readers can follow.

Published Paper Repository

Abstract (Plain Language)

The work proposes a faster 8-bit multiplier by combining Vedic multiplication (for parallel partial-product generation) with Brent-Kung adder stages (for efficient carry handling). The target is lower propagation delay without excessive hardware complexity.

1) Research Problem

In DSP and arithmetic datapaths, multiplication often limits clock speed. Traditional architectures can become slow because of carry propagation and deeper logic paths.

2) Proposed Idea

Use Urdhva Tiryagbhyam decomposition for multiplication and place Brent-Kung adder logic on key merge paths so carry depth is reduced where it matters most.

3) Methodology from the Paper

  1. Split both 8-bit operands into 4-bit halves.
  2. Compute four 4-bit Vedic products in parallel.
  3. Combine cross terms using a Brent-Kung driven adder stage.
  4. Merge middle bits with carry-aware addition.
  5. Resolve upper product bits through final adder stage.
  6. Produce the final 16-bit multiplication output.

4) Experimental Setup (Published)

  • Design entry: Verilog HDL.
  • Simulation/synthesis context in paper: Xilinx Vivado 14.7.
  • Comparison set: Array, Dadda, Wallace Tree, and existing Vedic multiplier variants.

5) Architecture in This Repo

  • Hierarchy from `vedic2bit` up to `vedic8bit`.
  • Prefix adder cells: `BlackCell`, `GrayCell`, `pg16`, `xor16`.
  • Exhaustive self-checking testbench for full 8-bit input space.

7) What This Means for Non-Experts

Lower delay means the arithmetic block can potentially run at a higher clock frequency. The proposed design seeks this speed gain while keeping logic depth and complexity controlled.
